仕組み

Ordito は IR（中間表現）・テンプレート契約・コレクション（ナビ）・生成エンジン の4つに分かれています。

設計の芯

  • 制約された多様性 — 枠は固定、中の本文は揺れてよい。IR フィールドの消失はバグ
  • 更新 ≠ 生成 — IR 更新は軽く頻繁に。HTML 生成は重く明示的に（「記載 → 反映」の二段確認）
  • すべて JSON、すべてスキル — 拡張はストアへの読み書きスキルとして足す

field_map とは

テンプレート契約はブロック型の全フィールドの行き先を field_map で定義します。未マップのフィールドは検証で検出されます。表示しないフィールドは "OMIT" と明示します。

INFO実装初期、IR にあった default 値が AI 出力から静かに脱落した事例が、この仕組みを作るきっかけになりました。

データの流れ

  1. 記載エージェントがスキルで IR を更新（generated: false・ビルドは走らない）
  2. 未反映検出ordito-detect-stale で updated_at > generated_at のページを一覧
  3. 反映ordito-generate で HTML を再生成し generated_at を押印
  4. 検証ordito-validate でスキーマ・field_map・出力を確認
