エラーハンドリング

API はエラーを HTTP ステータスと JSON ボディで返します。ボディには code と message が含まれます。認証エラーについては 認証 も参照してください。

エラーコード

ステータスコード意味
400invalid_requestリクエストの形式が不正です。
401invalid_keyAPI キーが無効、または失効しています。
403insufficient_scope必要な権限スコープがありません。
429rate_limitedレート制限に達しました。レート制限 を参照。
WARNING429 を受け取ったら、Retry-After ヘッダーの秒数だけ待って再試行してください。

推奨する対処

  1. 4xx はリクエスト側の問題。内容を修正して再送する。
  2. 5xx は指数バックオフで再試行する。
  3. 429 は Retry-After に従う。

復旧しない場合は サポート に request_id を添えて連絡してください。
